Outlier flagging

The point. Every blood test gets placed into one of the cluster profiles from the previous tab. Most land squarely inside one. A few don't — either they're stuck between two profiles, or they're a long way from every profile. Those are the tests worth a second look, and that's what this view surfaces.

What you're looking at. A ranked list of the least-confident tests, with the reason each was flagged. "Boundary case" means the test sits between two clusters — the model genuinely can't decide. "Multivariate outlier" means the test lands in one cluster but sits unusually far from that cluster's typical pattern, by a model-based distance, not just a single odd marker.

When it's interesting. A short list = most of the cohort fits the cluster profiles cleanly. A long list of boundary cases = your clusters overlap; the profile boundaries are fuzzy. A long list of multivariate outliers = the cohort has heavy tails and the Gaussian model is fighting them. Either way, the listed tests are the ones to start with.

How it works — technical detail

Two flagging rules. Both rules use outputs of the cluster model on the Clusters tab — nothing is re-fit here.

1. Boundary cases. The cluster model assigns each test a posterior probability per cluster (the probabilities sum to 1). We flag tests whose maximum posterior is below 0.7, meaning no single cluster claims more than 70% of the test's profile — the remainder is split between two or more clusters. The "Why investigate" line names the top two competing clusters and their probabilities.

2. Multivariate outliers. For each test we compute the squared Mahalanobis distance to its assigned cluster's mean — measured on the un-winsorised projection, so a record whose extreme value was capped for the clustering fit is still caught here. In the PCA-reduced space: d² = Σ (xᵢ−μᵢ)² / σᵢ². Under the model this follows a χ² distribution with df = n_cluster_dims degrees of freedom, so we flag tests whose d² exceeds the χ² critical value at p = 0.01 (scipy.stats.chi2.ppf(0.99, df)). This is an absolute threshold — if no test is genuinely unusual, none are flagged. The previous quantile-based rule (bottom 5% of log-likelihood) always flagged ~5% of tests by construction and has been retired.

Confidence and sorting. The Confidence column is the maximum posterior probability, expressed as a percentage. Rows are sorted ascending by Confidence so the tests the model is least sure about appear first.

Worked example — boundary case

Suppose a K = 3 cluster model assigned a test Tb the following posterior probability vector:

ClusterPosterior
Cluster 10.55
Cluster 20.40
Cluster 30.05

The probabilities sum to 1.00. The hard label is argmax → Cluster 1. The maximum posterior is 0.55, which is below the 0.7 threshold, so Tb is flagged as a boundary case.

The "Why investigate" line is built from the top two posteriors:

  • 55% Cluster 1 · 40% Cluster 2

The third cluster is omitted (only the two competing components matter for the investigation prompt). Confidence in the table is round(0.55 · 100) = 55%.

Worked example — multivariate outlier

Now consider a different test To that does land confidently in a cluster (max posterior > 0.7, so the boundary rule does not flag it). Suppose the cluster model kept three principal components and assigned To to Cluster 1, with:

  • To in PCA-reduced space: x = (4.5, −2.0, 1.5)
  • Cluster 1 mean: µ = (1.5, −0.5, 0.5)
  • Cluster 1 diagonal covariance: σ² = (0.6, 0.4, 0.5)

With diagonal covariance, the squared Mahalanobis distance reduces to a sum of squared z-scores:

Axis(xᵢ − µᵢ)²÷ σᵢ²Contribution
PC1(3.0)² = 9.00÷ 0.615.00
PC2(−1.5)² = 2.25÷ 0.45.625
PC3(1.0)² = 1.00÷ 0.52.00
Sum22.625

The χ² critical value at p = 0.01 with df = 3 is scipy.stats.chi2.ppf(0.99, df=3) ≈ 11.345. Since 22.625 > 11.345, To is flagged as a multivariate outlier: under the assumed Gaussian-mixture model, only ~1% of tests should sit this far from their assigned cluster's mean by chance.

PC1 dominates (15.00 of the 22.625), so To's anomaly is concentrated on the most explanatory axis — if PC1 captured, say, an overall metabolic-load score, this test is unusually far from its cluster on that axis.

Why the old 5% rule was retired

Previously the Outliers tab flagged the tests in the bottom 5% of log-likelihood within each cluster. That rule always flags ~5% of the cohort by construction: on a uniform cohort with no genuine outliers it still surfaces five of every hundred tests, which is misleading. The χ² rule above is an absolute threshold — if no test is genuinely unusual, none are flagged. On a cohort with many genuine outliers (e.g. heavy-tailed marker distributions), it flags more than 1%; on a clean Gaussian cohort, it flags close to 1%; on a uniform cohort with no outliers, it flags zero.

Assumptions and limitations.

  • The χ² threshold is calibrated under the Gaussian-mixture assumption. If a cluster's true within-cluster distribution is non-Gaussian (heavy-tailed, skewed), expected outlier rates may differ from the nominal 1%.
  • The 0.7 boundary threshold is a heuristic. Tighter (0.6) catches more boundary cases; looser (0.8) fewer.
  • Both rules inherit the assumptions of the cluster model itself: approximately linear structure (PCA), diagonal within-cluster covariance, and Gaussian-shaped clusters (GMM).
  • Diagonal covariance ignores within-cluster correlation between PCs; if real clusters are elongated along non-axis directions, the Mahalanobis distance under-states actual distance and outliers may be missed.
